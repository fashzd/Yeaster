# Yeaster — x402 alpha sales

Yeaster doesn't just trade its edge — it **sells** it. The agent's strongest graded
signal of the day is offered for sale over [x402](https://x402.org), the HTTP-native
micropayment scheme: a buyer pays a small USDT amount on BNB Smart Chain, the server
verifies the transfer **on-chain**, and only then releases the pick. No accounts, no
API keys, no custody — the paywall is a real settled transaction.

Off by default. Enable with `YST_X402=1`.

Code: `yeaster/api/routers/x402.py` (HTTP), `yeaster/execution/x402.py` (settlement),
`yeaster/brain/alpha.py` (the sellable signal), `scripts/buy_alpha.py` (buyer demo).

## What's for sale

`brain/alpha.py::daily_alpha()` selects the highest-composite graded candidate from
the latest commit record — the same signal the autonomous brain acts on. `teaser()`
returns a **locked preview**: posture, conviction, and the proof-chain hash, but **not**
the symbol or thesis. The full pick (symbol, thesis, levels) is released only after a
verified payment.

## The flow

```
buyer                          Yeaster API                         BNB Smart Chain
  │  GET /api/x402/alpha/teaser     │                                    │
  │ ───────────────────────────────▶  posture/conviction/proof (locked) │
  │  POST /api/x402/alpha  {}        │                                    │
  │ ◀─────────────────────────────── 402 Payment Required + requirements │
  │   send ≥ price USDT to pay_to ───┼───────────────────────────────────▶ (Transfer)
  │  POST /api/x402/alpha {payment_tx}│                                   │
  │                                  │  verify Transfer log on-chain ─────▶
  │                                  │  ◀──── amount ≥ price, to == pay_to │
  │ ◀─────────────────────────────── 200 { paid, amount_usd, alpha }     │
```

1. **Quote** — `GET /api/x402/alpha/teaser` returns the price, the recipient
   (`pay_to`), the asset (USDT), the network (bsc), and the locked teaser.
2. **402** — `POST /api/x402/alpha` with no `payment_tx` returns **HTTP 402 Payment
   Required** plus the `requirements` block (price, asset, network, pay_to, scheme
   `exact`, and a human `how` string).
3. **Pay** — the buyer sends `≥ price_usd` USDT to `pay_to` on BSC by any means.
4. **Redeem** — `POST /api/x402/alpha { "payment_tx": "0x…" }`. The server:
   - **anti-replay** — `is_tx_consumed(tx)` rejects a tx hash already redeemed (each
     payment unlocks once; the settlement log is the ledger);
   - **on-chain verify** — `verify_onchain_payment(tx, pay_to, price)` pulls the tx
     receipt over a BSC RPC, scans its ERC-20 `Transfer` logs for the USDT contract,
     and confirms a transfer to `pay_to` of `≥ price`;
   - on success, `record_alpha_sale(...)` appends to the settlement trail and the
     response carries the **full** `alpha` (symbol, thesis, levels).

Any failure (no tx, replayed tx, unverified transfer) returns **402** again with the
same requirements, so the flow is safely retryable.

## Verification properties

- **Real settlement.** The unlock is gated on a mined on-chain USDT transfer to the
  agent's wallet — not a signature or a claim. The price floor is enforced (`amount ≥
  price`); overpayment is recorded at its true amount.
- **Replay-proof.** A tx hash can be redeemed exactly once (`is_tx_consumed`); the
  settlement log in `data/` is the source of truth.
- **No custody / no accounts.** The buyer pays from their own wallet; the server only
  reads chain state.

## Status & trail

`GET /api/x402` reports `enabled`, scheme, asset, network, `price_usd`, `pay_to`,
`total_settled_usd`, and the recent `settlements`. The control room surfaces the
buy-alpha flow on the **/alpha** page.

## Configuration

| Var | Default | Effect |
|---|---|---|
| `YST_X402` | `0` | enable alpha sales + the x402 endpoints |
| `YST_X402_SECRET` | dev | HMAC key for the (optional) signed-receipt path |
| `YST_X402_PRICE_USD` | `0.10` | price in USDT to unlock the daily pick |
| `YST_X402_PAYTO` | agent wallet | recipient address payments are verified against |

The recipient defaults to the agent's own self-custody wallet, so alpha sales accrue
to the same address the agent trades from.
